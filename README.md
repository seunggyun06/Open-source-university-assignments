# Linux 명령어 top, ps, jobs, kill에 대해서 알아보자


---

##  1. top — 실시간 프로세스 모니터링

###  설명

`top` 명령어는 CPU, 메모리 사용량 및 실행 중인 프로세스를 **실시간으로 모니터링**하는 명량아입니다.

###  주요 기능

* CPU, 메모리 사용률 모니터링
* 프로세스 목록 실시간 표시
* 프로세스 우선순위 변경(renice)
* 프로세스 종료(kill)

###  실행 방법

```bash
top
```
**명령어를 입력 후 실행하면 된다.**

###  유용한 단축키

| 단축키   | 기능            |
| ----- | ------------- |
|  P  | CPU 사용량으로 정렬|
|  M  | 메모리 사용량으로 정렬|
|  N  | PID 순으로 정렬|
|  T  | Running Time으로 정렬|
|  R  | 오름차순과 내림차순 토글 변경|

##  2. ps — 프로세스 정보 출력

###  설명
`ps` 명령어는 현재 실행 중인 프로세스의 **스냅샷**을 출력합니다.

### 'ps -efl' 실행 결과 사진
![ps 사진](https://github.com/user-attachments/assets/78719a66-2318-4b2b-abc3-5e72ac706132)


###  주요 옵션

| 명령어      | 설명                      |
| -------- | ----------------------- |
| `ps`     | 현재 쉘의 프로세스 출력           |
| `ps -e`  | 시스템 전체 프로세스 출력          |
| `ps -f`  | 자세한(full) 정보 출력         |
| `ps aux` | 가장 많이 사용되는 BSD 형식 전체 목록 |

### ps 명령어 출력 항목
|ps 명령어| 설명|
| -------- | ----------------------- |
| USER (BSD), UID (System V) | 프로세스 소유자의 이름 |
| PID | 프로세스의 식별 번호 |
| PPID | 부모 프로세스의 PID |
| %CPU|CPU 사용 비율의 추정치 (BSD)|
| %MEM|Memory 사용 비율의 추정치(BSD)
|VSZ|K 단위 또는 페이지 단위의 가상 메모리 사용량|
|RSS|실제 메모리 사용량|
|TTY|프로세스와 연결된 터미널|
|S (System V), STAT (BSD)|현재 프로세스의 상태 코드|
|TIME|총 CPU 사용 시간|
|COMMAND|프로세스의 실행 명령행|
|STIME|프로세스가 시작된 시간 혹은 날짜|
|C (System V), CP (BSD)|짧은 기간 동안의 CPU 사용률|
|F|플래그|
|PRI|실제 실행 우선순위|
|NI|nice 우선순위 번호|

##  3. jobs — 백그라운드 작업 확인

###  설명

현재 쉘에서 실행 중인 **백그라운드 작업 목록**을 확인합니다.

###  사용 예시

```bash
jobs
```
**명령어를 입력 후 실행하면 된다.**

###  출력 예시

```
[1]+  Running   python server.py &
```
* [1]+ 첫 번째 열의 대괄호 안의 숫자는 잡 ID가 됩니다. 이 번호는 현재 셸에서 유효합니다. 또한 그 다음에 있는 +는 bg나 fg 명령 실행시 기본 인자로 사용된다는 의미입니다. -는 +로 표시된 프로세스가 종료시 기본값으로 사용될 프로세스를 의미합니다.
* Running 두 번째 열은 프로세스의 상태를 나타냅니다. 현재는 Running으로 실행중인 상태를 의미합니다.
* python server.py & 프로세스를 실행한 명령어를 나타냅니다.

###  상태 유형

| 상태          | 의미            |
| ----------- | ------------- |
| **Running** | 실행 중          |
| **Stopped** | 일시 중지(Ctrl+Z) |
| **Done**    | 완료됨           |

### jobs 명령어 옵션
| 옵션 | 의미 |
| ----------- | ------------- |
|-l 옵션 | 프로세스 ID와 함께 잡 목록을 출력합니다.|
|-n 옵션 | 마지막로 알림 이후 변경된 잡만 출력합니다.|
|-p 옵션 | 잡의 프로세스 ID만 출력합니다.|
|-r 옵션 | 실행 중인 잡만 출력합니다.|
|-s 옵션 | 중지된 잡만 출력합니다|

##  4. kill — 프로세스 종료

###  설명

`kill` 명령어는 프로세스에 **시그널(signal)**을 보내 종료 또는 제어합니다.

###  주요 시그널

| 시그널         | 번호 | 의미          |
| ----------- | -- | ----------- |
| **SIGTERM** | 15 | 정상 종료       |
| **SIGKILL** | 9  | 강제 종료       |
| **SIGSTOP** | 19 | 실행 중지       |
| **SIGCONT** | 18 | 중지된 프로세스 재개 |

###  사용 예시

```bash
kill 1234       # 정상 종료
kill -9 1234    # 강제 종료
kill -STOP 1234 # 중지
kill -CONT 1234 # 재개
```

### 'kill' 명령어 관련 기타 명령어

| 명령어          | 설명                     |
| ------------ | ---------------------- |
| **pkill**    | 프로세스 이름 기반 종료          |
| **killall**  | 동일한 이름의 모든 프로세스 종료     |
| **xkill**    | GUI 창을 클릭해서 종료(X11 환경) |
| **fuser -k** | 파일/포트를 사용하는 프로세스 종료    |
